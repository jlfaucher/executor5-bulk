What is OCube

OCube is an ooRexx application for manipulating standard text files. It can perform transformations on existing lines of the file as well as add, replace and delete entire lines from the file.

OCube performs these transformations by reading a procedure file of instructional statements. Each statement in the procedure file performs an action on the target text file or contains a runtime instruction. The syntax for each of these statements can be found in the section Procedure file statements.

Since OCube is an ooRexx program it works the same on all operating systems supported by ooRexx.

OCube has been designed so that the source code is easily modified or extended. If you have suggestions for extending the functionality of OCube or source code to contribute then send them to dashley@us.ibm.com. Each suggestion or source modification will be considered for inclusion into the next version of OCube.






Using OCube

OCube is an ooRexx application designed to be invoked from the command line. The following shows the syntax for invoking OCube.

 >>---rexx---ocube.rex---procfile---targetfile---+----------------+------->
                                                 +---backupfile---+

 The following options may appear in any order:

 >----+-------------+----+-------------+----+---------------+------------->
      +--- -DEBUG---+    +--- -PAUSE---+    +--- -VERBOSE---+

 >----+---------------------+--------------------------------------------><
      +--- -TRUNCINLINES ---+
procfile
The path/name of the OCube procedure file. This file contains the OCube procedure statements. If the procfile specified is QUEUE then the Rexx default queue will be used to obtain the procedure statements. If procfile is enclosed with curly brackets ( {xxx} ) then it will be treated as a single procedure statement to be executed against the targetfile.
targetfile
The path/name of the target text file. This is the file to be modified by OCube procfile statements.
backupfile
The path/name of the backup file to be created. If specified, OCube will write a copy of targetfile to this file before operating on it.
-DEBUG
OCube will produce debug information.
-PAUSE
OCube will pause after every procedure statement.
-VERBOSE
OCube will produce verbose output of the changes made to the target file.
-TRUNCINLINES
OCube will truncate space and tab characters from the end of all target file input lines prior to performing any changes.

File names may optionally be enclosed within double-quotes.

Examples

The following are some examples for invoking OCube.

 rexx ocube.rex myproc.txt c:\config.sys c:\config.bak -verbose

 rexx ocube.rex myproc.txt c:\config.sys -pause -truncinlines

 rexx ocube.rex {AddLine "REM my new line" after 'PATH='} c:\config.sys

 rexx ocube.rex queue "c:\My Documents\myfile.txt" -debug -verbose





Procedure File Statements

Procedure file statements perform transformations to the target file or modifiy the behavior of OCube. Each statement can perform single or multiple modifications to the target file.

The syntax of OCube procedure statements is flexible. Strings may be surrounded by either double or single-quotes. Statement options may appear in any order (but they may not be repeated within a single statement). Keywords in statements are case-insensitive.

Statements may appear on multiple lines within the procedure file. The line continuation character is a comma (,) and must appear as the last character on the line to be continued. Comment statements may not be inserted between continued lines.

The procedure file syntax supports three kinds of comment statements. All lines beginning with an asterisk (*) or double dashes (--) are considered to be comments and are ignored by OCube. Blank lines are also considered to be comment lines.

The syntax for each procedure file statement is listed below.

Statements

AddLine - add a line to the target file
AddString - add a string to a line in the target file
CommentLine - comment line(s) in the target file
DelLine - delete line(s) from the target file
DelString - delete string in line(s) in the target file
OnError - change the runtime error processing option
RepLine - replace line(s) in the target file
RepString - replace strings in line(s) in the target file

ADDLINE Statement

The ADDLINE statement conditionally adds a line to the target file.

Syntax Diagram

 >>---+---ADDLINE---+---'newline'----------------------------------------->
      +---AL--------+

 The following options may appear in any order:

 >----+-----------------------------+----+-------------------+------------>
      +---AFTER----+---'matchstr'---+    +---CASESENSITIVE---+
      +---BEFORE---+

 >----+---------------+----+------------+----+----------------------+----><
      +---ADDTOP------+    +---ANYPOS---+    +---RSDLM---'dlmstr'---+
      +---ADDBOTTOM---+
      +---DONTADD-----+
newline
Single or double-quoted string containing the line to be added to the target file.
AFTER
Add the newline after the line specified by matchstr.
BEFORE
Add the newline before the line specified by matchstr.
matchstr
Single or double-quoted string which will be used to identify position(s) in the target file where newline will be added.
CASESENSITIVE
Specifies that a case sensitive search will be performed using the matchstr. The default is a case insensitive search.
ADDTOP
If matchstr is not found in the target file then add newline to the top of the target file. This is the default. This option is ignored if matchstr is found.
ADDBOTTOM
If matchstr is not found in the target file then add newline to the bottom of the target file. This option is ignored if matchstr is found.
DONTADD
If matchstr is not found in the target file then do not add newline to the target file. This option is ignored if matchstr is found.
ANYPOS
Specifies that matchstr may appear anywhere in the line being searched. The default specifies that the matchstr must appear at the beginning of the line being searched.
dlmstr
Single or double-quoted string which specifies the delimiter to be used for environment string substitutions on the newline and matchstr. Only the first character of the string is significant. If no dlmstr is specified then no environment string substitution will be performed.

Examples

AddLine 'REM my new line' after 'PATH='

This example will add the specified new line after the first line found in the target file which contains the string 'PATH=' at the beginning of the line. If no matching line is found then the line will be added to the top of the target file.

AddLine 'REM my new line' before 'PATH=' addbottom anypos casesensitive

This example will add the specified new line before the first line found in the target file which contains the string 'PATH=' at any position of the line. If no matching line is found then the line will be added to the bottom of the target file. The case of the search string must match the existing case of the string within the line for a match to be considered true.

AddLine 'set mypath="c:\my documents"' after 'PATH=%path%' rsdlm '%'

This example will add the specified new line after the first line found in the target file which contains the string 'PATH=%path%' at the beginning of the line. If no matching line is found then the line will be added to the top of the target file. Before the search is performed the search string will be modified and the contents of the environment variable 'path' will be replaced within the search string.

ADDSTRING Statement

The ADDSTRING statement conditionally adds a string to an existing line or adds a new line to the target file.

Syntax Diagram

 >>---+---ADDSTRING---+---'newstring'---IN---'matchstr'------------------->
      +---AS----------+

 The following options may appear in any order:

 >----+----------------------------+----+-------------------+------------->
      +---AFTER----+---'srchstr'---+    +---CASESENSITIVE---+
      +---BEFORE---+

 >----+---------------+----+------------+----+----------------------+----><
      +---ADDTOP------+    +---ANYPOS---+    +---RSDLM---'dlmstr'---+
      +---ADDBOTTOM---+
      +---DONTADD-----+
newstring
Single or double-quoted string to be added to a target file line.
matchstr
Single or double-quoted string which will be used to identify position(s) in the target file where newstring will be added.
AFTER
Add the newstring after the string specified by srchstr. If srchstr is not specided then newstring will be added to the end of the line specified by matchstr. If matchstr is not found then a new line will be inserted per the ADDTOP, ADDBOTTOM or DONTADD option.
BEFORE
Add the newstring before the string specified by srchstr. If srchstr is not specided then newstring will be added to the beginning of the line specified by matchstr. If matchstr is not found then a new line will be inserted per the ADDTOP, ADDBOTTOM or DONTADD option.
CASESENSITIVE
Specifies that a case sensitive search will be performed using the matchstr. The default is a case insensitive search.
ADDTOP
If matchstr is not found in the target file then add newstring as a new line to the top of the target file. This is the default. This option is ignored if matchstr is found.
ADDBOTTOM
If matchstr is not found in the target file then add newstring as a new line to the bottom of the target file. This option is ignored if matchstr is found.
DONTADD
If matchstr is not found in the target file then do not add newstring to the target file. This option is ignored if matchstr is found.
ANYPOS
Specifies that matchstr may appear anywhere in the line being searched. The default specifies that the matchstr must appear at the beginning of the line being searched.
dlmstr
Single or double-quoted string which specifies the delimiter to be used for environment string substitutions on the matchstr, srchstr and newstring. Only the first character of the string is significant. If no dlmstr is specified then no environment string substitution will be performed.

Examples

AddString 'c:\bin;' in 'PATH=' after 'PATH=' dontadd

This example will add the specified string after the first line found in the target file which contains the string 'PATH=' after the string 'PATH='. If no matching line is found then no action is performed.

COMMENTLINE Statement

The COMMENTLINE statement conditionally inserts text at the beginning of lines in the target file.

Syntax Diagram

 >>---+---COMMENTLINE---+---'matchstr'---WITH---'cmtstring'--------------->
      +---CL------------+

 The following options may appear in any order:

 >----+-------------------+----+------------+---+----------------------+-->
      +---CASESENSITIVE---+    +---ANYPOS---+   +---RSDLM---'dlmstr'---+

 >----+-----------+------------------------------------------------------><
      +---FIRST---+
      +---LAST----+
      +---ALL-----+
matchstr
Single or double-quoted string which will be used to identify the target file line(s) to be commented.
cmtstring
The string which will be placed at the beginning of the line(s) identified by the matchstr.
CASESENSITIVE
Specifies that a case sensitive search will be performed using the matchstr. The default is a case insensitive search.
ANYPOS
Specifies that matchstr may appear anywhere in the line being searched. The default specifies that the matchstr must appear at the beginning of the line being searched.
dlmstr
Single or double-quoted string which specifies the delimiter to be used for environment string substitutions on the matchstr. Only the first character of the string is significant. If no dlmstr is specified then no environment string substitution will be performed.
FIRST
Comment only the first line found which matches the matchstr. This is the default. This option is ignored if matchstr is not found.
LAST
Comment only the first line found which matches the matchstr. This option is ignored if matchstr is not found.
ALL
Comment all lines matching matchstr. This option is ignored if matchstr is not found.

Examples

CommentLine 'COUNTRY=' with 'REM'

This example will comment the first line in the target file which contains the string 'COUNTRY=' at the beginning of the line by tranforming the line to 'REM COUNTRY=...'.

DELLINE Statement

The DELLINE statement conditionally deletes line(s) from the target file.

Syntax Diagram

 >>---+---DELLINE---+---'matchstr'---------------------------------------->
      +---DL--------+

 The following options may appear in any order:

 >----+-----------+----+----------------------+----+------------+--------->
      +---FIRST---+    +---RSDLM---'dlmstr'---+    +---ANYPOS---+
      +---LAST----+
      +---ALL-----+

 >----+-------------------+----------------------------------------------><
      +---CASESENSITIVE---+
matchstr
Single or double-quoted string which will be used to identify the target file line(s) to be deleted.
FIRST
Delete only the first line found which matches the matchstr. This is the default. This option is ignored if matchstr is not found.
LAST
Delete only the first line found which matches the matchstr. This option is ignored if matchstr is not found.
ALL
Delete all lines matching matchstr. This option is ignored if matchstr is not found.
dlmstr
Single or double-quoted string which specifies the delimiter to be used for environment string substitutions on the matchstr. Only the first character of the string is significant. If no dlmstr is specified then no environment string substitution will be performed.
ANYPOS
Specifies that matchstr may appear anywhere in the line being searched. The default specifies that the matchstr must appear at the beginning of the line being searched.
CASESENSITIVE
Specifies that a case sensitive search will be performed using the matchstr. The default is a case insensitive search.

Examples

DelLine 'PATH='

This example will delete the first line in the target file which contains the string 'PATH=' at the beginning of the line.

DelLine 'PATH=' all anypos casesensitive

This example will delete all the lines from the target file which contain the string 'PATH=' at any position of the line. The case of the search string must match the existing case of the string within the line for a match to be considered true.

DELSTRING Statement

The DELSTRING statement conditionally deletes a string in an existing line in the target file.

Syntax Diagram

 >>---+---DELSTRING---+---'srchstr'--------------------------------------->
      +---DS----------+

 The following options may appear in any order:

 >----+---------------------+----+-------------------+-------------------->
      +---IN---'matchstr'---+    +---CASESENSITIVE---+

 >----+------------+----+----------------------+-------------------------><
      +---ANYPOS---+    +---RSDLM---'dlmstr'---+
srchstring
Single or double-quoted string. This is the target string to be deleted in a target file line.
matchstr
Single or double-quoted string which will be used to identify position(s) in the target file where strings will be deleted. If matchstr is not specified then all lines in the target file will be searched for srchstr. If matchstr is not found then no action will be taken.
CASESENSITIVE
Specifies that a case sensitive search will be performed using the matchstr. The default is a case insensitive search.
ANYPOS
Specifies that srchstr may appear anywhere in the line being searched. The default specifies that the srchstr must appear at the beginning of the line being searched.
dlmstr
Single or double-quoted string which specifies the delimiter to be used for environment string substitutions on the srchstr and matchstr. Only the first character of the string is significant. If no dlmstr is specified then no environment string substitution will be performed.

Examples

DelString 'c:\bin;' in 'PATH='

This example will delete the specified string after the first line found in the target file which contains the string 'PATH='. If no matching line is found then no action is performed.

ONERROR Statement

The ONERROR statement modifies the OCube runtime error processing.

Syntax Diagram

 >>---ONERROR---+---STOP-------+-----------------------------------------><
                +---CONTINUE---+
STOP
Stop processing the OCube procedure file when an error condition is raised. This is the OCube startup setting.
CONTINUE
Continue processing the OCube procedure file when an error condition is raised.

Examples

OnError stop

This example will cause OCube to stop processing the procedure file when an error condition is raised. Under this condition the target file will not be modified in any way.

REPLINE Statement

The REPLINE statement conditionally replaces line(s) in the target file.

Syntax Diagram

 >>---+---REPLINE---+---'matchstr'---WITH---'newline'--------------------->
      +---RL--------+

 The following options may appear in any order:

 >----+-------------------+----+------------+---+----------------------+-->
      +---CASESENSITIVE---+    +---ANYPOS---+   +---RSDLM---'dlmstr'---+

 >----+-----------+------------------------------------------------------><
      +---FIRST---+
      +---LAST----+
      +---ALL-----+
matchstr
Single or double-quoted string which will be used to identify position(s) in the target file where newline will replace the existing line.
newline
Single or double-quoted string containing the line to replace matching line(s) in the target file.
CASESENSITIVE
Specifies that a case sensitive search will be performed using the matchstr. The default is a case insensitive search.
ANYPOS
Specifies that matchstr may appear anywhere in the line being searched. The default specifies that the matchstr must appear at the beginning of the line being searched.
dlmstr
Single or double-quoted string which specifies the delimiter to be used for environment string substitutions on the matchstr and newline. Only the first character of the string is significant. If no dlmstr is specified then no environment string substitution will be performed.
FIRST
If matchstr is found in the target file then replace only the first occurence of the matching line. This is the default. This option is ignored if matchstr is not found.
LAST
If matchstr is found in the target file then replace only the last occurence of the matching line. This option is ignored if matchstr is not found.
ALL
If matchstr is found in the target file then replace all occurences of the matching lines. This option is ignored if matchstr is not found.

Examples

RepLine 'PATH=' with 'PATH=c:\;c:\mypath;c:\bin'

This example will replace the first occurence of the line containing the 'PATH=' at the beginning of the line. The entire line will be replaced. If no matching line is found then no action is taken against the target file.

RepLine 'PATH=' with 'PATH=c:\;c:\mypath;c:\bin' last anypos

This is the same as the previous example except the the last line found in the target file which contains the string 'PATH=' at any position of the line will be replaced. If no matching line is found then no action is taken against the target file.

REPSTRING Statement

The REPSTRING statement conditionally replaces a string in an existing line in the target file.

Syntax Diagram

 >>---+---REPSTRING---+---'srchstr'---WITH---'newstring'------------------>

 The following options may appear in any order:

 >----+---------------------+----+-------------------+-------------------->

 >----+------------+----+----------------------+-------------------------><
      +---ANYPOS---+    +---RSDLM---'dlmstr'---+
srchstring
Single or double-quoted string. This is the target string to be replaced in a target file line.
newstring
Single or double-quoted string. This is the string to replace srchstr in target file line(s).
matchstr
Single or double-quoted string which will be used to identify position(s) in the target file where strings will be replaced. If matchstr is not specified then all lines in the target file will be searched for srchstr. If matchstr is not found then no action will be taken.
CASESENSITIVE
Specifies that a case sensitive search will be performed using the matchstr. The default is a case insensitive search.
ANYPOS
Specifies that matchstr may appear anywhere in the line being searched. The default specifies that the matchstr must appear at the beginning of the line being searched.
dlmstr
Single or double-quoted string which specifies the delimiter to be used for environment string substitutions on the matchstr, srchstr and newstring. Only the first character of the string is significant. If no dlmstr is specified then no environment string substitution will be performed.

Examples

RepString 'c:\bin;' with 'd:\bin;' in 'PATH='

This example will replace the specified string after the first line found in the target file which contains the string 'PATH='. If no matching line is found then no action is performed.

Known Problems

  1. Target files with embedded tab characters may cause problems for OCube. You may want to convert the tab characters to spaces before processing the file with OCube.
